QUESTION:

Assume a Computer design problem having following parameters: Main memory: 4KB, Direct mapped cache, Cache size: 32 lines, Block size: 8 Bytes. Computer program accesses (in order) memory locations: 7, 8, 24, 25, 256, 260, 280, 520, 524, 1024 (i) What is the hit ratio? (ii) What are the final cache contents? Assume that the first three blocks of RAM are already copied into cache.

ANSWER:

We are provided with direct mapped cache. In direct mapping, a particular block of memory is mapped to a particular line of cache. The formula to calculate line number is

Line number = Block address % number of lines

Number of lines ( given ) = 32

We will calculate line number for each memory location and then put them in the corresponding line number. If there already exists another block in that line, then it is replaced. A cache hit will occur, if the block to be inserted already exists in the cache, otherwise it is a miss.

Calculation of line number for accessed memory locations ( in order ):

Line number = Block address % number of lines

7 % 32 = 7 ( MISS )

8 % 32 = 8 ( MISS )

24 % 32 = 24 ( MISS )

25 % 32 = 25 ( MISS )

256 % 32 = 0 ( MISS )

260 % 32 = 4 ( MISS )

280 % 32 = 24 ( MISS )

520 % 32 = 8 ( MISS )

524 % 32 = 12  ( MISS )

1024 % 32 = 0 ( MISS )

Number of hits = 0

Hit ratio = 0 / 10 = 0, where 10 is the number of memory locations accessed

In this below picture, you can see how the blocks are inserted in different line numbers. I have only shown the line numbers that has blocks in them. As you can see, how 256 was inserted first at Line 0, but then its was replaced by inserting 1024, whose line number is also 0.

The final cache contents are: 1024, 260, 7, 520, 524, 280 and 25.
