Q2) Binary Search helps perform serach operation on a sorted list of elements.The list is divided into 2 parts from the mid point and this mid point is compared with the serached element.If mid point is > serached element then second half of the list is discarded otherwise the first half is discarded. This process is repeated till the element is found.

Linear search on the other hand does not require a sorted list of elements. It compares the first element with the searched element and moves to second element in the list if not found. For a linear serach an average of N/2 comparisions and a maximum of N comparisions are required, where N is the number of elements in the array. 

Binary serach on the other hand requires an average of log2(N) comparisions.

For suppose 1024 elements in array, Linear search requires average 1024/2=512 comparisions while Binary search requires log2(1024) = 10 comparisions.

Now suppose there are 1024*1024 = 1048576 elemments in the array. Linear search requires average 1048576/2=524288 comparisions while Binary search requires log2(1048576 ) = 20 comparisions.

So it can be easily concluded that Adavntage of Binary search over Linear search increases when searching large sorted arrays

Hence the answer is TRUE.
