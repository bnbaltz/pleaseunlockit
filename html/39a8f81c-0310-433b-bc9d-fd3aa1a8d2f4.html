Suppose you insert 125 integer keys into a hash table with an initial capacity of 25 and a load factor threshold of 2. The hash code is the key itself. Assume the function key mod table length is used to map a key to a table position. How many total units of work will be done by the time all keys are inserted if it takes one unit of work to do the mapping, one unit to check load factor against threshold, and one unit to insert an entry into a linked list? Assume a rehash doubles the table capacity, and that the load factor is checked AFTER an entry is inserted into its linked list. Ignore the work done to create an array and/or to initialize an array to nulls. Show your work. 

 

Suppose A and B are both binary search trees.  BST A has "m" nodes and BST B has "n" nodes.  Assume m is much greater than n.  We want to find the common itmes in BST A and BST B.  We will do an inorder traversal of BST A, appending items to an array as they are encountered.  Same thing happens for BST B to a second array.  Assume there is enough space to hold all the items appended.  We will then need to find the comment elements between the two arrays.  What is the worst case big O running time?

 

ANSWER:

Assume initial capacity (N)= 25.

Load factor = n/N=n/25=2

=> n=50. where n= number of elements in hash table.

Thus, need to insert 50 integers, before rehash.

  • Number of Units of work to insert an integer= 1(for mapping) +1(for inserting into linked list) =2

Total units of Work to insert first 50 integers=50x2=100 units of work

  • After inserting 50 integers, threshold is reached. Thus, rehash and capacity will be 50.

Re insert the 50 integers into resized hash table.

Total units of Work to re-insert the first 50 integers=50x2=100 units of work

  • Load factor= n/N=n/50=2. Then, n=100.

Need to insert 50 integers before rehash, since already hash table contains 50 integers.

Total units of Work to insert next 50 integers=50x2=100 units of work

  • After inserting 50 integers, threshold is reached. Thus, rehash and capacity will be 100.

Re insert the 100 integers that are already inserted into old hash table, into resized hash table.

Total units of Work to re-insert the 100 integers=100x2=200 units of work

  • Load factor= n/N=n/100=2. Then, n=200.

Need to insert 200 integers before rehash. But only, 25 integers are left to insert, and no more rehashes are required. Thus insert the remaining 25 integers int hash table.

Total units of Work to insert remaining 25 integers=25x2=50 units of work

Total units of work done to insert 125 integers= 100+100+100+200+50 =550 units of work
