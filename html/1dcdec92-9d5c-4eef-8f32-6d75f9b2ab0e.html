i. Mesh Topology

The mesh topology is a wired or wireless networking structure where computers and network devices are connected in an interconnected network. For N number of nodes in a network, the links are N(N-1)/2. The nodes relay data and sends out their own data. It is one of the most common types of network tolopogy as a transmissions are not interrupted even when one node goes down.

Advantages Disadvantages
Relays data even when one node goes down as the mesh distributes data with the other systems in the network. Installation is complex and difficult.
Reduces data transmission redundancy due to to high distribution over the network. Redundancy in connections may occur.
Can easily add extra nodes or systems. Implementation is costly in comparison to other topologies.
Extremey robust and consistent. Maintainence is high with this topology.
Does not have a central point of authority. Higher power requirement to run the entire network.

ii. Bus Topology

The bus topology connects all the systems - computers and networking devices - to one single networking cable called the "backbone." The data is carried by the cable and checks it chacks if the correct recipient machine is founcd by comparing the MAC or IP address and the cable then transmits the data to the system that matches.

Advantages Disadvantages
Optimum for small networks. Not ideal for networks with a large number of devices.
Adding and removing devices in this topology is easy due to its singular cable "backbone." Difficult to deal with individual system issues within the bus network.
The cost of implementation is comparitively small as the length of the cable is small among other things. Since all the devices are connected to one cable, damage to the cable can impar teh network.
The bus topolgy is easy to understand and navigate Rate of transmission of packets is slower in comparison.
Extremely easy to install and connect devices to a network. High probability for data or pack lost during transmission.

iii. Star Topology

The star topology is a network where all the systems in a network is connected to a central system which acts as a hub. All information in the network is first relayed to the hub and then proceeds to the intended recipient system. Thus each device in the network as connect with its own cable to the hub. Therefore for each system N, the number of cables required would be N.

Advantages Disadvantages
Reliable than the bus or ring topology. The whole network fails if the central hub fails.
Can add or remove systems from the network without disrupting the other connections. Requires more cables than bus topology
Since the systems are lined up to the hub, its easy to detect any faults that occur at a certian node.  Performance of the network depends on the central system or hub.
Requires only one port to connect to the central system. The central hub requires a lot of mantainence to ensure the network is connected without issues.
It is cost effective when compared to mesh topolgy. When compared to bus topology, the extra cables, central hubs etc. makes implementation and installation costly.

iv. Ring Topology

The ring topology has all the systems in the network connected to one another in the form of a ring. Each system is connected to two other systems in close proximity by cables. Repeaters are used to ensure data loss does not occur. 

Advantages Disadvantages
Packet loss is minimized as data flows in one direction. It is time consuming as the packets have to travel through all the nodes to reach the intented recipient system.
Since the systems are connected to each other in an orderly fashion, central hubs are not required. As the systems are conencted in a ring, if one node is removed or goes down, the entire network shuts down.
This topology is easy to manage as all systems are connected with only one cable. Since they depend on one cable, its hard to scale it to include many systems without increasing risk of network faliure.
All nodes have equal access to all the resources. All systems must be operational for data to be relayed and the network to function.
Can manage traffic better than bus topology. Troubleshooting in a ring topology is difficult.
