QUESTION:

Suppose you send a 100 Mb file to your friend at 5 km away. The channel data rate is 10 Mbps and the propagation speed is 2 x 10^8 ms^-1 . Stop-and-Wait ARQ is used in the transmission. Each frame carries 1500 bits of data plus 24 bits of overhead (header + trailer). ACK frame has a fixed size of 50 bits. The queuing and processing delay along the path can be neglected.  Use 1K = 1000, 1M = 1000K and 1G = 1000M. 

1. How many data frames will be sent? What are the sizes of the frames? 

2.  Assume there are 2 frames lost in every 100 frames sent, no frame lost if less than 100 frames sent, and the last frame is not lost. By using a timeout value of (10 * x) ms (where x=5), calculate the total time required for the process to complete.

ANSWER:

SOLUTION:

Senders and receivers window size is one in stop N wait protocol.

Given file(data) size= 100M bits.

Distance =5000metre.

Propagation speed =2*10^{8}m/s.

Frequency of channel=10Mbps.

Frame header size=24bits.

Data size for each frame without header=1500 bits

Acknowledgement size= 50 bits.

Propagation time = 5000/2 * 10 seconds

=25*10^{-6}seconds=25microseconds.

Round trip time=50 microseconds

And (a) Number of frames required for sending entire file = 100*10^{6}/1500

=66,667 frames will be required to send entire file.
