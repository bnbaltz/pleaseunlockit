QUESTION:

Consider the following relation:

a) Given the previous extension (state), which of the following dependencies may hold in the above relation? If the dependency cannot hold, explain why by specifying the tuples that cause the violation.

i. A ? B,  ii. B ? C,  iii. C ? B,  iv. B ? A,  v. C ? A

 

b)  Does the above relation have a potential candidate primary key (not a superkey)? If it does, what is it? If it does not, why not?

ANSWER:

a)

i. A->B. Cannot hold. Eg : Tuple 1 has 10->b1, and 2# has 10->b2. This is the violation

ii. B->C. This holds true. Eg : b1->c1 and b3->c4. There are no violation case

iii. C->B does not holds true. Eg  : c1->b1 and c1->b4. This is the violation.

iv. B->A does not holds true. Eg : b1->10 and b1->13. 

v. C->A does not holds true. Eg : There is violation of c1 tuple. c1->10, c1->11, c1->13

 

b) This table does have a primary key, (B,C) because there is a dependency from B->C. There will always be a unique relation from B->C.
