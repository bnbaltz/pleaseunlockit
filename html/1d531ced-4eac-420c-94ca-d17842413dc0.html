Answer :

A) Size of frame = 1500(data) + 24(header, trailer) = 1524bits

Frames=   

Frames = (100*1000)/1500 = 66.66 = 67 as frames can not be in decimal.

B) Time taken to send 1 frame =Transmission Time + Propogation Time

Transmission Time = frame size/ bandwidth = 1524/(10*1000) = 0.1524sec

Propogation Time = distance of channel / velocity of channel = (5 *1000)/ 2*1000000000 = 0.000025sec

Time taken to get acknowledgement by reciever = T.T. + P.T. of ack.

T.T. of ack = ack frame size/bandwidth = 50/(10*1000)

= 0.0005sec

P.T of ack = P.T. of frame = 0.000025sec

Total time to get acknowledgement of first frame= time taken to send 1 frame + time taken to send acknowledgement = 0.1524+ 0.000025+0.0005+0.000025 = 0.15295sec

C) Total time to send 100Mb data = Time taken to send 1 frame * no. Of frames = 0.15295 * 67 = 9.7888sec

D) The receiver will check for the redundancy using CRC technique. If he finds some bits corrupted in the frame he will not send the acknowledgement to sender. Therefore the entire frame is required to be resent to the receiver.
