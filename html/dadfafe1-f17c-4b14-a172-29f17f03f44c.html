Ans (d) Sender mantains a buffer where it keep the recently sent data. It will discard the data and refill the buffer only when it receives the ACK for that data.If any frame is corrupted then two case can arise-

1) Receiver will not send any ACK. 

For this case sender will thought that either the data is lost while propagating or data was corrupted. Sender maintains a timer known as timeout timer. Once the timeout occur sender will resend the data present in the buffer pointed by the current senders window.

2) Receiver sent NACK.

In this case receiver will send Not Acknowledged frame to sender telling sender that your frame is corrupted. So sender will resend the data for NACK frame.
