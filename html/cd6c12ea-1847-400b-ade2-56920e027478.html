QUESTION:

Research how to implement anti-spoofing and directed broadcast filters on some type of router?

ANSWER:

Cisco routers after IOS version 12.3 allow directed broadcast to be disabled by issuing the following command in enable mode.
no ip directed-broadcast

You can also create an advanced access control list (ACL)to filter directed ip broadcasts from specific ranges.  This can be used if you have some services running remotely that do need to perform directed broadcasts, but want to deny them for all other hosts.

To prevent spoofing attacks, in which a host attempts to use an internal IP address from outside the network, you create access lists to prevent packets from entering through the external interface with a source destination that would be on the inside network.  This can be done with a basic ACL.  For example, if your internal network was the 192.168.1.0/24 network, you first create the following ACL

ip access-list ext ingress-antispoof

deny ip 192.168.1.0 0.0.0.255 any

And then apply it to your inbound WAN interface (We'll assume Serial 0/0)

int s0/0#ip access-group ingress-antispoof in

To be thorough, you would need to apply this access list to every interface other than that has an interface IP address in that range.
