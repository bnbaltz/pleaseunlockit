REA model for Fixed Asset System:

The procedure involves requesting for a fixed asset by manager to purchase department clerk who sends the request to supplier. Receiving department receives the fixed asset from the vendor (supplier). Upon receiving the fixed asset, cash disbursement post check to supplier. There are lots of account, journal updating done here which need not be shown in REA. Also, asset maintenance and asset disposal records and procedure need not be shown in REA.

• The Resources are Fixed Asset, Cash.

• Events are Request Asset, Receive Asset, Disburse Cash

• Agents are Manager, Supplier, Receiving Clerk, Cash Disbursement Clerk. ( need not show Purchasing Clerk as the agent for Request Asset as Purchasing Clerk in turn sends the request to Supplier).

The Cardinality is almost same as the explanation given in the solution 8. It is not mentioned that suppliers are giving materials on credit, hence for each Receive Asset, there has to be one and only one Disburse Cash event and cannot have 0 Disburse Cash. The association between the events and the agents here is that, for each event, there has to be only one agent and each agent can perform 0 or many of those events. So, all the cardinalities here are M, 0:1, 1

All the above entities can be considered as a Table, but Manager, Receiving Clerk and Cash Disbursement Clerk will be considered under Employee Table.
